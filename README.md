# Система разграничения доступа для информационной системы автоматизации пунктов проката

## Описание проекта

Данный проект представляет собой прототип информационной системы для пунктов проката оборудования с реализованными механизмами разграничения доступа на основе ролевой модели RBAC (Role-Based Access Control) и технологий построчной фильтрации RLS (Row-Level Security) в СУБД PostgreSQL.

### Основные возможности

- Ролевая модель доступа: Клиент, Менеджер, Администратор
- Построчная фильтрация данных: каждый пользователь видит только разрешённые записи
- Защита персональных данных в соответствии с Федеральным законом N 152-ФЗ "О персональных данных"
- Региональное ограничение: менеджеры видят данные только своего филиала
- Автоматическое тестирование политик безопасности через CI/CD

### Сценарии использования

| Роль | Что видит | Что может делать |
|------|-----------|------------------|
| Клиент | Только свои договоры аренды | Просматривать свои аренды и каталог оборудования |
| Менеджер | Аренды и клиентов своего филиала (без паспортных данных) | Создавать, редактировать и закрывать договоры аренды |
| Администратор | Все данные системы, включая паспортные данные | Полный доступ, аудит, управление пользователями |

## Технологии

- PostgreSQL 15 или выше
- Row-Level Security (RLS)
- GitHub Actions (CI/CD)
- Bash scripting
